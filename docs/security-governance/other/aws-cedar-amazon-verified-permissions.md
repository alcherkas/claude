# AWS Cedar & Amazon Verified Permissions: formally-verified authorization

> **Excerpt from the original:** Cedar is a policy language purpose-built for application-level authorization, with a functional, per-policy-independent evaluation model and four formally-established properties: default-deny, forbid-overrides-permit, order-independent, and deterministic (peer-reviewed, ACM OOPSLA 2024). Amazon Verified Permissions runs the authorization infrastructure as a managed service; applications call APIs that evaluate Cedar policies in managed policy stores.

**License:** Cedar: Apache-2.0 (open source) · Amazon Verified Permissions: managed AWS service

**Relevance to securing & governing AI coding agents:** A formally-verified authorization engine usable as the decision point for agent actions, and a pluggable backend in the Microsoft Agent Governance Toolkit. AWS reportedly added a natural-language-to-Cedar capability in Bedrock AgentCore (March 2026), not deeply verified here.
