# Guardrails AI: composable input/output validation for LLMs

> **Excerpt from the original:** An open-source framework running Input/Output Guards composed from Hub Validators that intercept LLM inputs/outputs at runtime to detect, quantify, and mitigate risks (toxicity, PII, prompt injection) and generate structured data. Validation failures trigger configurable on-fail actions: reask, fix, filter, refrain, noop, or exception.

**License:** Apache-2.0

**Relevance to securing & governing AI coding agents:** Content/output validation rather than authorization, and general-purpose rather than coding-agent-specific — a complement to, not a substitute for, policy-as-code authorization at the tool-call layer.
