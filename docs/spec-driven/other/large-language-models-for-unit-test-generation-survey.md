# Large Language Models for Unit Test Generation

arXiv

> **Excerpt from the original:** "the community has converged on an iterative generate-validate-repair paradigm, which treats test generation not as a one-shot task but as a debugging process" — deterministic execution feedback (compiler errors, runtime exceptions, test failures) is captured from external toolchains and injected back into the prompt for self-correction, "a key architectural component connecting probabilistic models with deterministic quality requirements." The survey also groups autonomous testing agents such as TestForge and SocraTest.

**Relevance to TDD / feedback-sensor harnesses:** The academic framing of the generate-validate-repair loop that underpins the whole feedback-sensor category. Evidence note: this convergence claim rests on a single survey scoped to unit-test generation and was the run's only non-unanimous claim (2-1) — *medium confidence*.
