# The Virtual Monorepo Pattern: full system context across 35 repos for Claude Code

> **Excerpt from the original:** Owen Zanzal describes giving Claude Code "full system context across 35 repos" by assembling them into a single **virtual monorepo** — a parent workspace that aggregates many independent repositories and layers context files over them, so the agent can reason across the whole system in one session.

**Relevance to multi-repo context engineering:** A representative **independent re-derivation** of the meta-repo umbrella, aimed specifically at AI-agent context. It is one of roughly six practitioners (Zanzal, Groff, Kampl/Anyline, Uy, Soporan, Pant) who arrived at the same single-coordination-point-over-many-repos structure without citing each other — which is what gives the *approach* (if not any single coined name) real cross-practitioner weight. Treat as a practitioner account, not a vendor-blessed pattern.
