# meta: the meta-repo / virtual monorepo tool

> **Excerpt from the original:** `meta` (mateodelnorte/meta, ~2015, ~2.2k★) manages many repositories as if they were one — a parent "meta repo" plus a `.meta` manifest and a `meta` CLI that runs git and shell commands across all child repos. Its tagline captures the idea directly: *"Why choose many repos or a monolithic repo… when you can have both?"*

**Relevance to multi-repo context engineering:** This is the canonical tooling behind the **"meta-repo / virtual monorepo"** umbrella — the one multi-repo topology term with genuine **multiple-independent backing**, and the one that *predates the AI-agent era* (so it is not an AI-context coinage). The June 2026 research pass found ~6 practitioners independently re-deriving this structure to give coding agents whole-system context (see the Virtual Monorepo write-up), which is why "meta-repo" is the strongest of the otherwise practitioner-coined topology patterns.
