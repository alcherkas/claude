# About this wiki

## What this is

This site is a curated, cross-linked, searchable index of authoritative sources
on engineering AI coding agents. It began as five research briefs and was
expanded into a browsable wiki where every referenced source has its own page
with a short annotation, an excerpt, and a link to the original.

## Provenance

The five themes started as research briefs (curated link libraries with
annotations) covering context engineering, security & governance, orchestration,
spec/test-driven development, and evaluations & observability. Each brief
prioritized primary, recent (2024–2026) material from five sources:

- **Anthropic** — Claude Code, agent engineering, harnesses
- **Google** — ADK, A2A, SAIF, Gemini tooling
- **Microsoft / GitHub** — Copilot, Agent Framework, Azure guidance
- **ThoughtWorks** — Technology Radar, harness engineering, sensors
- **Martin Fowler** — Exploring Gen AI, harness engineering

…plus high-value practitioner write-ups and academic (arXiv) references.

## Usage & attribution

!!! note "Excerpts and links, not reproductions"
    Each source page shows only the curator's **annotation**, a short
    **excerpt** (≤ ~75 words) to aid recall, and a **link to the original**.
    Full article text is **not** reproduced here. Copyright in each source
    remains with its original publisher; please read and cite the original.

If you are a rights holder and would like an excerpt adjusted or removed, please
open an issue on the [repository](https://github.com/alcherkas/claude).

## How it's built

The wiki is generated by a small Python tool (`tools/wikigen`) and published with
[MkDocs Material](https://squidfunk.github.io/mkdocs-material/):

1. **parse** — extract every link, its annotation, and its section from the
   source briefs.
2. **manifest** — de-duplicate to one page per URL (cross-referencing themes
   that share a source).
3. **fetch** — politely download each source into a local-only archive
   (rate-limited, robots-aware, with retries; arXiv via its API).
4. **build** — render the public pages (annotation + excerpt + link) and a build
   report, then MkDocs builds the static site.

The full downloaded text is kept **only on the maintainer's machine** (a
git-ignored `archive/`) for private reading and search; it is never published.

## Caveats

- Some sources are paywalled, block automated access, or are JavaScript-rendered;
  those pages show the annotation and link only (and a Wayback snapshot when one
  exists).
- Links rot over time. The build report tracks dead links so they can be
  re-checked.
- Excerpts are auto-selected and occasionally grab a lead that isn't the article's
  thesis — always prefer the original.
