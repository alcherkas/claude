# Beyond Self-Talk: a survey of multi-agent communication structures

> **Excerpt from the original:** This 2025 survey (arXiv:2502.14321) maps how LLM agent fleets are structured and coordinated. It enumerates **five communication architectures** — **Flat** (decentralized peer-to-peer), **Hierarchical** (layered strategic oversight), **Team** (specialized groups), **Society** (norm-governed), and **Hybrid** — and **three turn-taking strategies**: sequential **One-by-One**, concurrent **Simultaneous-Talk**, and **Simultaneous-Talk-with-Summarizer** (concurrency consolidated by a dedicated summarizer agent). It distinguishes **static vs. dynamic** orchestration, where dynamic schemes **activate/deactivate agents at runtime** (e.g., DyLAN's LLM ranker deactivates low-performing agents). It also flags that traditional all-to-all communication **does not scale** as agent count and task complexity rise.

**Relevance to agentic orchestration & workflows:** The academic taxonomy underneath the "swarm" vocabulary — it gives names and structure to *fleet* topology (flat/hierarchical/team/society/hybrid), turn-taking, and the static-vs-dynamic membership distinction that ThoughtWorks' "agent teams vs. coding agent swarms" framing gestures at. Pairs with the [collaboration-mechanisms survey](https://arxiv.org/abs/2501.06322) and grounds the scaling argument that motivates hierarchical decomposition ([SWARM+](swarm-plus-hierarchical-agent-fleet-scaling.md)). *Evidence: high-confidence — architecture/turn-taking/static-vs-dynamic taxonomy verified unanimous 3-0, cross-corroborated by a second independent survey (June 2026). A survey taxonomy is descriptive and stable, but it catalogues patterns rather than measuring which work best.*
