MCP Apps

Servers return an interactive UI that renders right inside the chat — not just text.

Why use them

  • Context preserved — the UI lives in the conversation; no tab switching.
  • Two-way data — the app calls server tools; the host pushes fresh results back.
  • Host integration — delegate actions (e.g. "schedule this") to tools the user already connected.
  • Sandboxed — runs in an isolated iframe; can't touch the host page, cookies, or storage.

Where they shine

  • Explore data — "sales by region" → a clickable map you drill into, instead of a wall of numbers.
  • Configure with many options — a deploy form with validation & defaults, not a 10-question back-and-forth.
  • Rich media — pan/zoom a PDF, rotate a 3D model, or preview generated images inline.
  • Live monitoring — a dashboard of metrics or logs that keeps updating without re-asking.
  • Multi-step review — triage issues or approve expenses one by one with buttons and saved state.
